FUNDAMENTALS OF THE WEB

LESSON 1: THE BASICS OF THE WEB AND HTML
Introduction

There are at least five important subjetcs to be studied as basic elements on the web: the web itself, the code language HTML, the repository URL, the protocol HTTP and proper web aplications. First of all the web - or world wide web.

The World Wide Web

The World Wide Web is a collection of HTML documents, which stands for Hypertext Markup Language. The documents are all linked to one another throught Hyper Links. All kinds of file can exist on the web, music, video, images - HTML documents can refer to anything.

There are four main actors in the web: users, computers + browsers, the internet itself and servers. The user put up an internet adress on his browser and it makes a request via the internet to the servers, using the HTTP protocol. The servers respond with files displaced by the browser. Servers are computers which host files, and the HTTP is the main protocol used on internet communications. Web documents, as said before, are written in a language called HTML, the next topic.

HTML - Hyper Text Markup Language

HTML or Hyper Text Markup Language is the language used to write documents placed on the web. It is made of four components: text content, or the text that can be seen; Markup, or what the text looks like; References to other documents; and Links to other pages. Text content is plain text, while markup is used to mark texts up.

HTML Markup

Texts are marked up using tags. Usually an element is made of an opening tab, a closing tab and contents, for example:

Ex: <name>contents</name> ----- This is called an element

In an element the contents inside the opening and closing tags are marked up with the effects of that tag. There are several kinds of tags, which one with its own markup effect on text, like the <b> bold tag, <em> emphasis - italics tag, Anchor tags <a>, used to make links, image tags <img>, among others. There are also tags with attributes <tag attr = "value">, which have values. It is important to say that image tags are void tags, that donÂ´t need a closing tab.

Whitespace

All that is put in HTML turn in a single space. If a user writes a text in multiple lines, it appears in a single line, in whitespace. It is necessary to force multiple lines using tags. The break tag <br> is the first way. It is a void tag to be used after any line the users wants a break. The other way is with paragrarh tag <p>, which is not a void tag. The main difference between those two tags is that <br> is an inline tag, while <p> is a block one.

Inline and Block elements

Block and inline tags have different behaviour. Inline elements fit within a line, sometimes ending it, like <br> tag. Block tags make and an invisible box around the text, that can have height and width. Examples of inline elements are: <a>, <span>, <br>, <img>, <strong>. Some examples of block elements: <div>,<p>, <form>.

Container elements

Containers are elements that contain stuff inside them. There are two container tags: <span> and <div>. Span is an inline element, while div is a block one. They both can use CSS, which is a language to give styles to documents.

Computers and stupidity

It is good to remember that computers are stupid, and that is not a bad thing to say. They are stupid because they only interpret instructions literaly.

HTML, CSS and Javascript

HTML files are the structure of a webpage, CSS files are the style of a page, and Javascript files are the interactive components of a page. These languages define the rules for how webpages behave.

HTML, Structure and CSS: tree-like structure

Webpages have branched structures like trees (tree-like structure). Branches are formed in coding, from big elements to small ones. The same structure may result in different looking webpages, depending on the style imprinted by CSS. It is important to remember that HTML and CSS are both languages. HTML, CSS and the DOM

The DOM is the tree-like structure of a page. HTML is the language of the web. It has a specific syntax and rules. The basic word in the language of HTML is tag. The browser turns tags into elements, forming trees. It knows how to do it because of the DOM

The DOM or Document object module is a form to represent and interact with elements in HTML. The tree-like structure observed in developers tools section of the browser shows a DOM tree that the browser builds from a HTML document.

Each HTML tag creates an element in the DOM that browsers use to display the page. The content of a tag can be text, empty or even another tag. Every tag has a type or name that defines the element that will be created. Also tags can have attributes with values. With tags HTML documents define the structure and content of a page.

To define the style of a page it is necessary to use another language: CSS. CSS allows the use of specific syntax and rules to change how elements look on the page (fonts size, color, background, orders, positions, etc.)
Ex: It is possible with CSS to state that all <h1> elements that belong to the class "article" must be dark blue.

To tie structure and style together the HTML document can provide the browser with information on what style to use and where to get it.

LESSON 1: INTRODUCTION TO PROGRAMMING
Computer Science

Computer science is about solving problems. It works by breaking a problem into smaller pieces and then precisely and mechanically describing a sequence of steps that can be used to solve each piece. The task is to do that computer follow the steps and solve the problem by itself.

Computer

Machines are usually designed to do just one thing, like a toaster. The universal machine, in other hand, was imagined to do whatever the programmer told it to do, an universal problem solver. Computers are universal machines. Programs are used to tell a computer what to do. The power in this machine is that it can be programmed to do any sort of computation, the programmer just has to figure out how to program the order.

Programs

Programs tell computers what to do. The program must be a very precise sequence of steps. It is necessary to give instructions in a certain order. The power of the computer is that it can execute billions of instructions in a few seconds, following the sequence of steps very quickly. The program tells the computer which steps to take, and it follows. In order to design a program it is necessary a language to "speak" with the computer.

Programming languages - Python

Programs need to be designed in a language that can be understood by computers. As said before, computers are stupid, they only understand literal orders. Natural languages have syntatic and semantic problems that create certain difficulties with literal interpretations. One of them is ambiguity - the same sentence can be interpreted in many ways in natural languages. Another one is verbosity - natural languages require many words to describe a complicated situation. That is why a programming language is necessary. the programmer must be sure that the computer interprets the program the exact same way he wanted and programming languages can describe complicated situations in just a few lines of code.

Admiral Grace Hopper (1906-1992), for example, was a pioneer in computer sciences and writing computer languages. She wrote one of the first computer languages, called COBOL, for a long time the most used one. She was also the one of the first to create a compiler, a program that makes programs (at that time many thought computers could only do arithmetic).

There are many different languages for programming computers. Python is one that is going to be learned in this course. It is a high-level language to write programs. A high-level language means that instead of the program running directly on the computer, the programs written in python are the input to the python interpreter (which itself is a program), which runs on the computer.

Interpreter

An interpreter is a program that runs others programs written in a certain programming language, interprets and executes them, by running a program in the language that the computer can understand directly - machine language. The python interpreter does that with programs written in python language. The python interpreter can only interpret sentences in python language that it already previously knows. Python has a grammar, like natural languages, in order be possible to communicate - a whole syntactic set of rules to write sentences.

The difference between an interpreter and a compiler, like Admiral Grace Hopper built, is that a compiler does all the work at once, and them runs the new program, whereas on an interpreter the work is done at the same time.

Grammar

To create a language is necessary a basic set of rules, a grammar - the computer language grammar must be able to surpass the natural language syntax and semantic problems. John Backus, the lead designer of the fortran computer language for IBM in the fifties, helped to create the Backus-Naur notation form, in order to be able to precisely describe the language, in a way that is very simple and concise. It works with concepts of non-terminal, terminal and derivation to make valid expressions.

The Backus-Naur notation form start always with a non-terminal, that throught derivation, following the rules, reaches a terminal, constructing a valid expression. Python uses the Backus-Naur notation form to its grammar.

Python grammar and expressions

Python grammar is much stricter than normal language rules. The code must match the language exactly to get a correct syntax. If it does not, then an error message appears.

An expression is something that has a value, a valid sentence constructed with correct grammar rules. A Python expression is a valid Python statement. For example: print 3 + 5 is a valid expression, but print 3 + 5 + is not.

Mutation

Lists supports mutation, different from strings. By this process a value of a list can be changed after created. When a variable is assigned to a new string, it donÂ´t change the old one, just assign the variable to a new one. The plus operator also creates a new string, and does not change the old one. In lists the elements itself can be changed. This is called mutation.

Ex:
p = ['H','e','l','l','o']
p[0] = 'Y'.
This code changes the value 'H' in the list to 'Y'.

Mutation is specially important when another variable is introduced. For example:
Considering the last list created, the code q = p makes the two variables refer to the same list. The code q[4] = '!' changes a value referred in a list referred by q. Lists are mutable objects, while strings are immutable ones.

Draw turtle

We will learn about classes with functions that draw shapes. It is demonstrated a function that through drawing multiple squares can create a circle. The first step to drawing multiple squares is to draw a single square. To do that something called turtle can be used in python. Inside the python standard library there is a file called turtle, with lowercase t, and inside that file is a class, a class Turtle, with uppercase T. A class is like a package box that put things really well together and allows us to write code like turtle.Turtle(), as if we are calling a function. In fact, we are calling a function, named init (stands for initialize), which is defined inside the class Turtle. It creates space in memory for a new instance or a new object of the class Turtle. We can create an instance called Shinji using turtle. Shinji can now access all the rest of the methods there are inside class Turtle, like forward and right.

A class is like a blueprint of a building. A blueprint contains certain pieces of information about the building, for example, indicating the fact that it has a size and a color. It is also known that the same blueprint can be used to build multiple different buildings - a yellow office building or a blue residential one. A class Turtle, like a blueprint, indicates the fact that turtle has movements, and from this class, or blueprint, multiple instances can be created, like shinji, an instance of the class Turtle. These Individual instances of the class Turtle will contain information about their sizes, their colors or movements, for example.

There is an important difference between functions and classes. When we used functions previously, we wrote code like webbrowser.open(), but when we wanted to create instances of the class Turtle, we wrote code that said turtle.Turtle(). In the first case, all we are doing is calling a function, but in the second case, the init function defined inside of class turtle is the function that gets called, and it creates or initializes space inside memory for a new instance, in this case called shinji. Even though it feels like we are calling functions in both cases, not all functions are created equal. In the first case all we did was call a function, in the second case, the class called the function init, to create space for its instance, that not existed before.

LESSON 3: Object Oriented Programming - OOB
Object Oriented Programming: a first look

Object Oriented Programming can be seen as a programming model paradigm based on the manipulation of "objects". An object is like a thing that can perform certain tasks - it is a structure that contain data and procedures, which can be created and used by the programmer. Computer programs can be made by the interaction of objects. Programming languages that use object-oriented programming are often class-based, which means that objects are instances of classes. Classes can be thought as representations of a type of object, a plan or a bluerprint from which the objects are created. A great number of the most used programming languages today (often multi-paradigm) supports this programming paradigm, including python.

Utility of Object Oriented Programming

Object oriented programming is a valuable tool for programming projects. It permits a more efficient method of software-development, mainly because of three factors: modularity, extensability and reusability.

OOB is a modular programming paradigm, because it provides a way to develop a program by constructing separated units - a way to separate the duties of the programmer in software development. Modularity improves software maintainability, since parts of the system can be changed or updated without the need of large scale changes.

OOB is also extensible. Objects can be extended to include new attributes, not previously defined in the class they belong to. This can increase the powers of the programmer, to go beyond the previously stablished classes and add new things.

OOB permits an efficient way to reuse code. Objects can be reused within and across applications. This enables a faster software development with lower costs, since OOB languages come with a rich library of objects and the code created for a project can be reused for future ones.

When used properly these factors together enables the creation of a higher-quality software, since they permit that more time and resources be used in the software verification.

Classes and objects

One of the most important linguistic distinctions in object-oriented programming is between classes and objetcs. A simple question may enlight us in this matter: what is the difference between a dog and a german sheperd? Dogs are like classes, which the objects - german sheperds, belong to. In that way, classes indicate certain elements of objects, and objects belong to certain classes. Many objects can be build from a class. New vocabulary words

  • Class: the class keyword is used to define a class. Classes can be thought like blueprints. Classes can have both data and methods.
  • Instance: Multiple instances can be created in a class, such as star_wars and annie_hall. When we create an instance, the class constructor gets called.
  • Constructor: This is essentialy the init method inside the class. It is here that we initialize all of the data associated with the instance.
  • Self: we can think of self as itself or the instance in question. When the instance star_wars is being created, self is star_wars.
  • Instance variables: All the variables associated with a specific instance are called instance variables. These are unique to an object, and can be accessed using the self keyword inside the class and the instance name outside the class.
  • Instance method: all the functions inside the class that are associated with a specific instance and have self as the first argument are instance methods

Inheritance

Classes can inherit attributes from other classes. In that way, code can be reused, leading to less work. To show how this works, we will construct two classes: Parent and Child. The class child will inherit some attributes from the class Parent. The inheritance method is used in class Child. The syntax to do that follows: class Child(Parent): By doing so, the class Child will inherit all the attributes that are made public by the class Parent. To do that, the new class will use the old one init method.

When the code for child runs, the init method inside child will be called, after that the constructor for class parent will be called and runs. Then the rest of code for class child runs.

Code example: 

class Parent():
    def __init__(self, last_name, eye_color):
        print("Parent Constructor Called")
        self.last_name = last_name
        self.eye_color = eye_color


class Child(Parent):
    def __init__(self, last_name, eye_color, number_of_toys):
        print("Child Constructor Called")
        Parent.__init__(self, last_name, eye_color)
        self.number_of_toys = number_of_toys

billy_cyrus = Parent("Cyrus", "blue")

#print(billy_cyrus.last_name)

miley_cyrus = Child("Cyrus", "Blue", 5)

print(miley_cyrus.last_name)
print(miley_cyrus.number_of_toys)

Method inheritance and overriding

Inheritance can also be used to inherit methods. To do that, we will create a method in the class Parent from the previous example, show_information - to show information about the instance. Because class Child inherits from class Parent, this new method will be inherited, even though this method is not explicitly defined in class Child. We can create another show_info() method in the Class child, to go over all other attributes; This new method will override the previous one, and be used also by the class Parent. This is the ability of a subclass, like Child, to override a method inherited from other class. It is called in programming Method Overriding.
